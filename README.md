# Synchronization

It is recommended to synchronize your changes before submitting them.

During synchronization, the application:

- downloads changes made by other translators;
- uploads your local modifications;
- refreshes the translation list.

If a translation has already been changed on the server, it will automatically become **unapproved** and should be reviewed again.

<img width="1149" height="350" alt="image" src="https://github.com/user-attachments/assets/5c0e773b-c66c-432c-8449-94776a22baeb" />

---

# Submitting Changes

Once everything has been reviewed, click the **Submit Changes** button.

Only **approved** translations are sent to the server.

Unapproved translations remain in the application.

After a successful submission, approved translations are applied to the local files, so they are no longer considered modified.

# Recommended Workflow

1. Select a file.
2. Correct the Russian text if necessary.
3. Add or edit the English translation.
4. Approve the completed translations.
5. Synchronize with the server.
6. Review approved translations again if new changes were received.
7. Submit your approved translations.

---

# Tips

- Synchronize frequently.
- Review approved translations before submitting them.
- Save your work locally if you are offline.
- After synchronization, some translations may become unapproved again if another translator has already modified them. This is expected behavior.
